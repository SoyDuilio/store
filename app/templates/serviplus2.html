Pagos Integrados + Split Payment

💳 Funcionalidad Crítica para el Negocio

Esta es la funcionalidad más importante de la Opción Pro. Sin ella, el modelo de negocio basado en comisiones es inviable.

¿Qué es?

Un sistema de pagos integrado donde el cliente paga una sola vez en la plataforma, y el dinero se divide automáticamente entre SERVI-PLUS (comisión) y el profesional (pago del servicio).

Flujo de Pago

Cliente paga S/150
→
Niubiz/Culqi procesa
→
Split automático:
S/22.50 → SERVI-PLUS
S/127.50 → Profesional

Ejemplo Práctico

Caso Real

Servicio de Electricista

Costo del servicio: S/150

Comisión SERVI-PLUS (15%): S/22.50

Pago al electricista (85%): S/127.50

Proceso:

  1. Cliente ingresa tarjeta en SERVI-PLUS
  2. Paga S/150 (ve el monto total)
  3. Sistema automáticamente:
    • Retiene S/22.50 en cuenta de SERVI-PLUS
    • Deposita S/127.50 en cuenta del electricista
    • Genera comprobante para ambos

Sin Pagos Integrados (Plan Esencial)

❌ Problemas sin esta funcionalidad:

  • Cliente y profesional coordinan pago directo (efectivo/transferencia)
  • SERVI-PLUS pierde control de la transacción
  • Profesionales pueden "saltarse" la plataforma después del primer contacto
  • Difícil o imposible cobrar comisiones
  • Sin trazabilidad de pagos
  • Disputas difíciles de resolver

✓ Ventajas con pagos integrados:

  • 100% de comisiones capturadas automáticamente
  • Control total del flujo de dinero
  • Profesionales dependen de la plataforma
  • Trazabilidad completa
  • Protección contra fraudes
  • Comprobantes automáticos (SUNAT)

Tecnología Utilizada

En Perú, las dos opciones más confiables son:

Opción A: Niubiz (BCP)

  • Ventaja: Respaldado por el BCP, alta confianza
  • Comisión: ~3.5% + IGV por transacción
  • Soporte: Local, en español

Opción B: Culqi

  • Ventaja: Startup peruana, muy usado, API moderna
  • Comisión: ~3.99% + IGV por transacción
  • Soporte: Excelente, especializado en startups
# Ejemplo simplificado de Split Payment con API def process_payment(total_amount, professional_id): commission_rate = 0.15 # 15% para SERVI-PLUS commission = total_amount * commission_rate professional_amount = total_amount - commission # Llamada a API de Niubiz/Culqi payment = create_payment({ "total": total_amount, "split": [ {"account": "serviplus_account", "amount": commission}, {"account": professional_account, "amount": professional_amount} ] }) return payment

Registro Dual de Usuarios

¿Qué es?

La plataforma maneja dos tipos de usuarios completamente diferentes con perfiles, funcionalidades e interfaces distintas.

Tipo 1: Cliente (Consumidor de Servicios)

👤 Perfil de Cliente

Funcionalidades:

  • Buscar y solicitar servicios
  • Chat con profesionales
  • Pagar por servicios
  • Calificar y dejar reseñas
  • Ver historial de servicios contratados

Datos de perfil:

  • Nombre completo
  • Teléfono
  • Email
  • Direcciones guardadas
  • Métodos de pago

Tipo 2: Profesional (Proveedor de Servicios)

🔧 Perfil de Profesional

Funcionalidades:

  • Recibir notificaciones de trabajos disponibles
  • Aceptar o rechazar solicitudes
  • Chat con clientes
  • Confirmar servicios completados
  • Ver historial de trabajos y ganancias
  • Gestionar disponibilidad y zonas de cobertura

Datos de perfil:

  • Nombre completo y foto
  • DNI (verificado)
  • Certificados/licencias (para electricistas, gasfiteros)
  • Portfolio de trabajos (fotos)
  • Servicios que ofrece
  • Zonas de cobertura (distritos)
  • Disponibilidad horaria
  • Cuenta bancaria para recibir pagos
  • Rating promedio
  • Cantidad de trabajos completados

Arquitectura Técnica

Interfaces Diferenciadas

Vista del Cliente

Dashboard del Cliente

  1. Buscar servicio → Categorías (Plomería, Electricidad, etc.)
  2. Crear solicitud → Describir problema, subir fotos, indicar ubicación
  3. Ver ofertas → Profesionales interesados aparecen
  4. Chat → Conversar con el profesional seleccionado
  5. Pagar → Ingresar tarjeta y confirmar
  6. Calificar → Dejar reseña después del servicio
Vista del Profesional

Dashboard del Profesional

  1. Notificación → "Nuevo trabajo de electricidad en Miraflores"
  2. Ver detalles → Descripción, fotos, ubicación, pago estimado
  3. Aceptar/Rechazar → Decide si toma el trabajo
  4. Chat → Coordina detalles con el cliente
  5. Confirmar trabajo → Marca como completado
  6. Recibir pago → Dinero llega automáticamente a su cuenta

Proceso de Verificación de Profesionales

Para garantizar calidad y seguridad:

  1. Registro inicial: Llena formulario con datos personales
  2. Verificación de identidad: Sube foto de DNI (verificado con RENIEC API)
  3. Certificaciones: Para electricistas/gasfiteros, certificado del colegio profesional
  4. Entrevista virtual (opcional): Videollamada breve con admin
  5. Periodo de prueba: Primeros 5 trabajos son supervisados
  6. Aprobación: Si mantiene rating >4.0, queda activo permanentemente

Sistema de Cupones

¿Qué es?

Un sistema de descuentos estratégicos para atraer nuevos clientes, incentivar compras y fidelizar. El administrador controla todo desde el panel.

Tipos de Cupones

A) Cupón de Bienvenida

🎁 BIENVENIDO20

Descuento: 20% en el primer servicio

Tope: Máximo S/50 de descuento

Uso: Solo una vez por usuario nuevo

Objetivo: Reducir fricción en la primera compra

B) Cupón de Referido

👥 JUAN2024

Mecánica:

  • Cliente antiguo (Juan) comparte su código personal
  • Nuevo cliente usa código: recibe 15% descuento
  • Cliente antiguo (Juan): recibe S/20 de crédito en su cuenta

Objetivo: Crecimiento viral, cliente trae cliente

C) Cupón Promocional Temporal

📅 DOMINGOS10

Descuento: 10% de descuento

Válido: Solo los domingos

Objetivo: Aumentar demanda en días de baja actividad

D) Cupón Estacional

🎉 VERANO2025

Descuento: S/30 fijos en servicios de limpieza

Válido: Enero - Marzo 2025

Objetivo: Promocionar servicios específicos por temporada

Configuración Técnica

# Estructura de un cupón en base de datos class Coupon: code = "BIENVENIDO20" discount_type = "percentage" # o "fixed_amount" discount_value = 20 # 20% o S/20 max_discount = 50 # tope en soles min_purchase = 50 # compra mínima S/50 valid_from = "2025-01-01" valid_until = "2025-12-31" usage_limit_per_user = 1 # cuántas veces por usuario total_usage_limit = 1000 # uso total antes de expirar current_usage_count = 0 services_allowed = ["all"] # o ["plomería", "electricidad"] active = True # Aplicación del cupón def apply_coupon(total, coupon_code): coupon = get_coupon(coupon_code) if not validate_coupon(coupon): return {"error": "Cupón inválido o expirado"} if total < coupon.min_purchase: return {"error": f"Compra mínima: S/{coupon.min_purchase}"} # Calcular descuento if coupon.discount_type == "percentage": discount = total * (coupon.discount_value / 100) discount = min(discount, coupon.max_discount) # aplicar tope else: discount = coupon.discount_value final_total = total - discount return { "original": total, "discount": discount, "final": final_total }

Sistema de Rating y Reseñas

¿Qué es?

Un sistema de calificación bidireccional donde clientes califican a profesionales y viceversa. Crea confianza, transparencia y accountability.

Cliente Califica a Profesional

⭐ Calificación del Cliente

Estrellas generales: 1-5 estrellas

Categorías específicas:

  • Puntualidad (llegó a tiempo?)
  • Calidad del trabajo (resolvió el problema?)
  • Comunicación (fue claro y amable?)
  • Limpieza (dejó todo ordenado?)

Comentario de texto: Opcional, público

Fotos del resultado: Opcional (antes/después)

Profesional Califica a Cliente

👤 Calificación del Profesional

Estrellas generales: 1-5 estrellas

Categorías específicas:

  • Claridad en la solicitud (explicó bien el problema?)
  • Respeto (trato amable?)
  • Pago puntual (pagó sin problemas?)

Comentario: Opcional, privado (solo admin lo ve)

Objetivo: Identificar clientes problemáticos

Sistema de Reputación del Profesional

# Cálculo de rating promedio class Professional: rating_avg = 4.7 # Promedio ponderado total_reviews = 143 rating_breakdown = { 5: 120, # 120 reseñas de 5 estrellas 4: 18, 3: 3, 2: 1, 1: 1 } # Algoritmo ponderado (reseñas recientes pesan más) def calculate_weighted_rating(reviews): total_weight = 0 weighted_sum = 0 for review in reviews: age_days = (today - review.created_at).days # Reseñas de últimos 30 días pesan x1.5 if age_days <= 30: weight = 1.5 # Reseñas de últimos 90 días pesan x1.2 elif age_days <= 90: weight = 1.2 # Reseñas viejas pesan x1.0 else: weight = 1.0 weighted_sum += review.stars * weight total_weight += weight return weighted_sum / total_weight

Push Notifications en Tiempo Real

¿Qué es?

Notificaciones instantáneas que reciben los profesionales cuando hay una nueva solicitud de servicio. La velocidad de respuesta es el diferenciador clave vs. Facebook/clasificados.

⚡ Objetivo: Notificar en menos de 30 segundos

Desde que el cliente publica una solicitud hasta que el profesional la ve en su dispositivo: <30 segundos

Tecnología Utilizada

1. WebSockets (cuando están en la app)

Si el profesional tiene la PWA abierta, recibe notificación instantánea vía WebSocket.

2. Push Notifications (cuando NO están en la app)

Si el profesional no tiene la app abierta, recibe notificación nativa del sistema operativo (iOS/Android).

3. SMS como fallback (opcional)

Para solicitudes urgentes, se puede enviar SMS adicional (costo extra por SMS).

Tipos de Notificaciones

Para Profesionales

Notificaciones que reciben:

  • Nueva solicitud: "⚡ Nuevo trabajo de electricidad en Miraflores - S/150"
  • Cliente envió mensaje: "💬 Mensaje de María: ¿A qué hora puedes venir?"
  • Pago recibido: "💰 Pago de S/127.50 depositado en tu cuenta"
  • Nueva reseña: "⭐ Recibiste una reseña de 5 estrellas"
Para Clientes

Notificaciones que reciben:

  • Profesional aceptó: "✓ Juan aceptó tu solicitud de electricidad"
  • Profesional en camino: "🚗 Juan está en camino (15 min)"
  • Mensaje del profesional: "💬 Juan te envió un mensaje"
  • Servicio completado: "✅ Servicio completado. Por favor califica a Juan"

Configuración de Notificaciones

Los usuarios pueden controlar qué notificaciones reciben:

  • ✓ Nuevas solicitudes (profesionales no pueden desactivar esta)
  • ✓ Mensajes del chat
  • ✓ Pagos y transacciones
  • ✓ Reseñas y calificaciones
  • ○ Promociones y ofertas (opcional)

Geolocalización Avanzada

¿Qué es?

Sistema inteligente que encuentra profesionales cercanos al cliente, considerando distancia, tráfico de Lima y zonas de cobertura que el profesional definió.

📍 Problema que resuelve:

En Lima, nadie quiere viajar 2 horas por un trabajo. La geolocalización asegura que solo se notifique a profesionales que realmente pueden atender la zona del cliente en tiempo razonable.

Componentes del Sistema

1. Cliente Define Ubicación

Al crear solicitud, el cliente:

  • Ingresa dirección manualmente, o
  • Usa GPS del dispositivo (si da permiso), o
  • Selecciona de sus direcciones guardadas

2. Profesional Define Zonas de Cobertura

En su perfil, el profesional indica:

  • Distritos que atiende (ej: Miraflores, San Isidro, Barranco)
  • Radio máximo de distancia (ej: hasta 15 km)
  • Opción de "acepto trabajos fuera de mi zona por + S/20 de viaje"

3. Matching Inteligente

Cuando cliente publica solicitud:

  1. Sistema busca profesionales cuya zona de cobertura incluye la ubicación del cliente
  2. Calcula distancia real (no línea recta, sino por rutas)
  3. Considera tráfico típico de la hora (API de Google Maps)
  4. Ordena profesionales por:
    • Distancia (más cerca primero)
    • Rating (mejor calificados primero)
    • Disponibilidad (libres primero)
  5. Notifica a los top 5-10 profesionales más adecuados

Vista del Cliente: Mapa en Tiempo Real

Experiencia del Usuario

Una vez aceptado el trabajo:

El cliente puede ver en un mapa:

  • 📍 Su ubicación (punto azul)
  • 🚗 Ubicación del profesional en camino (punto en movimiento)
  • ⏱️ Tiempo estimado de llegada (actualizado en tiempo real)

"Juan está a 12 minutos de tu ubicación"

Tecnología

  • PostGIS: Extensión de PostgreSQL para consultas geoespaciales eficientes
  • Google Maps API:
    • Geocoding (dirección → coordenadas)
    • Distance Matrix (calcular distancias reales)
    • Directions API (rutas con tráfico)
# Consulta SQL con PostGIS para encontrar profesionales cercanos SELECT p.id, p.name, p.rating_avg, ST_Distance( p.location::geography, ST_SetSRID(ST_MakePoint(-77.0428, -12.0464), 4326)::geography ) / 1000 AS distance_km FROM professionals p WHERE 'electricidad' = ANY(p.service_types) AND p.available = TRUE AND ST_DWithin( p.location::geography, ST_SetSRID(ST_MakePoint(-77.0428, -12.0464), 4326)::geography, 15000 -- 15km radius ) ORDER BY distance_km, p.rating_avg DESC LIMIT 10;

Chatbot con Inteligencia Artificial

¿Qué es?

Un asistente virtual disponible 24/7 que responde preguntas frecuentes de clientes, reduciendo carga de atención al cliente humano.

Preguntas que Responde

🤖 Tipos de consultas:

  • "¿Cuánto cuesta un electricista?"
  • "¿Cuánto tiempo demoran en responder?"
  • "¿Qué incluye el servicio de limpieza?"
  • "¿Cómo pago?"
  • "¿Los profesionales están verificados?"
  • "¿Qué zonas de Lima cubren?"
  • "¿Puedo cancelar una solicitud?"

Ejemplo de Conversación

Chat en vivo

Cliente: Hola, cuánto cuesta un electricista?

Chatbot: ¡Hola! 👋 Los precios de electricistas varían según el trabajo:

  • Instalación de tomacorriente: S/80-120
  • Reparación de tablero: S/150-250
  • Instalación de luminaria: S/60-100

El precio exacto te lo dará el profesional después de ver el problema. ¿Te gustaría solicitar un electricista ahora?

Cliente: Sí, pero necesito uno urgente

Chatbot: Perfecto! 🚀 Los profesionales suelen responder en menos de 30 segundos. Para crear tu solicitud urgente, necesito que me des estos datos:

  1. ¿Cuál es el problema exacto?
  2. ¿En qué distrito estás?

Capacidades Avanzadas (Opción Premium)

En el plan Premium, el chatbot puede:

  • Diagnóstico preliminar: "Mi luz prende y apaga" → IA sugiere que es problema eléctrico de urgencia media
  • Estimación de precio: Basado en descripción del problema
  • Creación de solicitud guiada: El chatbot ayuda a llenar el formulario conversacionalmente

Escalamiento a Humano

Si el chatbot no puede responder:

  • Ofrece opción: "¿Quieres hablar con una persona?"
  • Transfiere a WhatsApp o chat con admin humano
  • Guarda historial de conversación para contexto

Tecnología

  • LLM API: Claude (Anthropic) o GPT-4 (OpenAI)
  • Prompts personalizados: Entrenados con FAQs específicas de SERVI-PLUS
  • Base de conocimiento: Documentos con info de servicios, precios, políticas
